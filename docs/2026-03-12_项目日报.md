## 🔍 问题与解决方案汇总

### 1. TypeScript 编译错误

**问题：**
```
The types of 'debug.exitCode' are incompatible between these types.
Type 'number | null' is not assignable to type 'number | undefined'.
```

**原因：** TypeScript 类型定义不匹配

**解决方案：**
```typescript
interface MoveResponse {
  debug?: {
    exitCode?: number | null;  // ✅ 允许 null
  };
}
```

### 2. 引擎启动失败 - Exit Code 127（多次）

#### 2.1 缺少执行权限

**错误：**
```
Spawn /var/user/engine/Rapfi22 eacces
```

**原因：** Vercel Linux 系统文件默认没有执行权限

**解决方案：**
```bash
git add --chmod=+x engine/Rapfi22
git commit -m "fix: 给 Rapfi22 添加 git 执行权限"
```

Git 提交模式从 `100644` → `100755`

---

#### 2.2 缺少依赖库 libtbb.so.2

**错误：**
```
/var/task/engine/Rapfi22: error while loading shared libraries: 
libtbb.so.2: cannot open shared object file
```

**原因：** Rapfi 引擎依赖 Intel TBB 并行计算库

**解决方案：**
- ✅ 上传 `libtbb.so.2` 到 Vercel
- ✅ 设置 `LD_LIBRARY_PATH` 环境变量

---

#### 2.3 系统库版本冲突

**错误：**
```
Inconsistency detected by ld.so: dl-call-libc-early_init.c: 37: 
Assertion `sym != NULL' failed!
```

**原因：** 上传的系统库（`libc.so.6` 等）与 Vercel Ubuntu 系统版本冲突

**解决方案：**
- ❌ 删除所有系统库文件
- ✅ 只保留专用库 `libtbb.so.2`
- ✅ 使用 Vercel 系统库

```bash
git rm --cached engine/libc.so.6 engine/libdl.so.2 ...
```

---

#### 2.4 GLIBC 版本不兼容

**错误：**
```
/var/user/engine/Rapfi22: /lib64/libm.so.6: version `GLIBC_2.29' not found
/var/user/engine/Rapfi22: /lib64/libstdc++.so.6: version `GLIBCXX_3.4.26' not found
```

**原因：** Vercel 系统库版本太旧（Ubuntu 18.04），Rapfi 需要更新的 glibc

**解决方案：**
- ✅ 上传兼容版本的 `libm.so.6` 和 `libstdc++.so.6`
- ✅ 强制添加到 Git（即使 .gitignore 排除）

```bash
git add -f engine/libm.so.6 engine/libstdc++.so.6
```

### 3. API 调用无限循环

**问题：** AI 落子后无限调用 API，Console 重复输出

**原因：** `useEffect` 依赖项包含 `gameState` 整个对象

```typescript
// ❌ 错误
}, [gameState, gameMode, difficulty, isGameOver]);

// 每次 setGameState 都会触发新的 useEffect
// 导致无限循环
```

**解决方案：** 使用 `useRef` 跟踪玩家轮换

```typescript
const lastPlayerRef = useRef<Player>('black');

useEffect(() => {
  const isWhiteTurn = gameState.currentPlayer === 'white';
  const wasBlackTurn = lastPlayerRef.current === 'black';
  
  // 只有当黑→白切换时才调用 AI
  if (gameMode === 'pve' && isWhiteTurn && wasBlackTurn && !isGameOver) {
    lastPlayerRef.current = 'white';
    // 调用 API...
  }
}, [gameState.currentPlayer, gameMode, difficulty, isGameOver]);
```

### 4. AI 不走棋（状态不更新）

**问题：** API 返回了着法，但棋盘没有更新，状态显示"白方回合（AI）"

**原因：** `setGameState` 使用回调函数时闭包捕获了旧状态

```typescript
// ❌ 错误
setGameState(prev => makeMove(prev, data.move.x, data.move.y));
// prev 可能是旧状态
```

**解决方案：** 添加详细调试日志，确保状态正确更新

```typescript
setGameState(prev => {
  const newState = makeMove(prev, data.move.x, data.move.y);
  console.log('New game state:', {
    currentPlayer: newState.currentPlayer,
    isChanged: newState !== prev,
    lastMove: newState.lastMove
  });
  return newState;
});
```

### 5. AI 返回非法坐标

**问题：** Rapfi 返回的坐标 `(7,7)` 已经被玩家占了

```javascript
Move validation: {
  x: 7, y: 7,
  isValid: false,     ❌ 位置不合法
  cellValue: 'black'  ❌ 这个位置是玩家刚下的黑棋
}
```

**根本原因：** API 传给 Rapfi 的棋盘数据不包含玩家最新落子

**解决方案：** 添加备用 AI 机制

```typescript
if (!isValid) {
  console.warn('Rapfi returned invalid move, falling back to built-in AI');
  workerRef.current?.postMessage({
    type: 'GET_BEST_MOVE',
    board: gameState.board,
    player: 'white',
    difficulty,
  });
  return;
}
```

### 6. Gomocup 协议调用错误（最终问题）

**问题：** Rapfi 持续返回已占用的坐标

**原因：** 使用了错误的命令格式

**错误的调用方式：**
```
board 15        ❌ 错误命令
black           ❌ 错误格式
turn 7,7        ❌ 这是对方落子后才用的
ai              ❌ 不需要，BOARD 后自动返回
```

**正确的 Gomocup 协议：**
```
START 15        ✅ 初始化棋盘大小
BOARD           ✅ 开始接收棋盘
7,7,1           ✅ x,y,颜色 (1=黑 2=白)
8,8,2
DONE            ✅ 发送完毕，AI 自动返回着法
```

**解决方案：** 重写 API 使用标准 Gomocup 协议

```typescript
// 1. START - 初始化棋盘
engine.stdin.write('START 15\n');

// 2. BOARD - 开始接收棋盘状态
engine.stdin.write('BOARD\n');

// 3. 发送所有落子 (格式：x,y,颜色 1=黑 2=白)
const moves = getMoveListWithColor(board);
for (const move of moves) {
  const color = move.color === 'black' ? 1 : 2;
  engine.stdin.write(`${move.x},${move.y},${color}\n`);
}

// 4. DONE - 发送完毕，AI 自动返回着法
engine.stdin.write('DONE\n');
```
