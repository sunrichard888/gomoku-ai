```typescript
export interface GameStats {
  totalGames: number;      // 总局数
  wins: number;            // 胜场
  losses: number;          // 负场
  draws: number;           // 平局
  winStreak: number;       // 当前连胜
  maxWinStreak: number;    // 最大连胜
  byDifficulty: {          // 按难度统计
    easy: { wins: number; losses: number; draws: number };
    medium: { wins: number; losses: number; draws: number };
    hard: { wins: number; losses: number; draws: number };
    expert: { wins: number; losses: number; draws: number };
  };
  lastUpdated: number;     // 最后更新时间戳
}
```

#### 2. 核心功能函数

**战绩管理**：
- `createInitialStats()` - 创建初始战绩数据
- `loadStats()` - 从 localStorage 加载战绩
- `saveStats()` - 保存战绩到 localStorage
- `recordGameResult()` - 记录游戏结果（自动更新连胜等统计）
- `calculateWinRate()` - 计算胜率

**关键逻辑**：
```typescript
export function recordGameResult(
  stats: GameStats,
  result: 'win' | 'loss' | 'draw',
  difficulty: string
): GameStats {
  const newStats: GameStats = {
    ...stats,
    totalGames: stats.totalGames + 1,
    lastUpdated: Date.now(),
  };

  // 更新胜负平统计
  if (result === 'win') {
    newStats.wins += 1;
    newStats.winStreak += 1;
    newStats.maxWinStreak = Math.max(newStats.maxWinStreak, newStats.winStreak);
  } else if (result === 'loss') {
    newStats.losses += 1;
    newStats.winStreak = 0;  // 失败重置连胜
  } else {
    newStats.draws += 1;
    newStats.winStreak = 0;  // 平局也重置连胜
  }

  // 更新按难度统计
  const diffKey = difficulty as keyof typeof newStats.byDifficulty;
  if (newStats.byDifficulty[diffKey]) {
    if (result === 'win') {
      newStats.byDifficulty[diffKey].wins += 1;
    } else if (result === 'loss') {
      newStats.byDifficulty[diffKey].losses += 1;
    } else {
      newStats.byDifficulty[diffKey].draws += 1;
    }
  }

  return newStats;
}
```

## 💡 设计决策

### 为什么只记录 PvE 模式？
- 避免 PvP 模式刷数据（两人可以故意刷胜率）
- PvE 模式更有挑战性，战绩更有意义
- 保持战绩系统的公平性

### 为什么平局也重置连胜？
- 平局不算胜利，连胜应该中断
- 符合大多数游戏的连胜规则
- 鼓励玩家追求真正的胜利

### 为什么使用 localStorage 而不是后端？
- 项目定位为单机游戏
- 无需用户系统
- 简单快速，零服务器成本
- 未来可扩展到后端（数据结构已设计好）
