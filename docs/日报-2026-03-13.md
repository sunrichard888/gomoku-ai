### 1. 落子音效系统 🔊
**问题**：游戏无音效，体验单薄

**解决方案**：
- 创建 `lib/audio.ts` 音频管理器
- 使用 Web Audio API 生成音效（无需外部音频文件）
- 实现三种音效：
  - **落子音**：短促的"嗒"声（800Hz→300Hz 快速衰减）
  - **胜利音**：欢快 ascending 音阶（C-E-G-C）
  - **平局音**：低沉 descending 音阶
- 添加音效开关按钮，状态持久化到 localStorage

**关键代码**：
```typescript
// lib/audio.ts
playPlace() {
  const oscillator = this.audioContext.createOscillator();
  const gainNode = this.audioContext.createGain();
  oscillator.frequency.setValueAtTime(800, now);
  oscillator.frequency.exponentialRampToValueAtTime(300, now + 0.05);
  gainNode.gain.exponentialRampToValueAtTime(0.01, now + 0.1);
}
```

### 2. 落子动画效果 ⭕
**问题**：棋子直接出现，无视觉反馈

**解决方案**：
- 添加 `AnimatedStone` 接口跟踪动画状态
- 实现缩放动画：`scale: 0 → 1.05 → 1.0`（弹性效果）
- 使用 `requestAnimationFrame` 每帧更新
- 动画时长约 0.5-0.7 秒

**技术难点**：
- Canvas 绘制需响应状态更新
- 避免循环依赖（`drawBoard` vs `useEffect`）
- 最终方案：状态更新自动触发 React 重绘

**关键代码**：
```typescript
// components/Board.tsx
const [animatedStones, setAnimatedStones] = useState<AnimatedStone[]>([]);

// 动画循环
useEffect(() => {
  const animate = () => {
    setAnimatedStones(prev => 
      prev.map(stone => ({
        ...stone,
        scale: Math.min(stone.scale + 0.03, 1.05),
      })).filter(stone => Math.abs(stone.scale - 1) >= 0.02)
    );
  };
  requestAnimationFrame(animate);
}, []);
```

### 3. 游戏存档系统 💾
**问题**：刷新页面后游戏进度丢失

**解决方案**：
- 使用 localStorage 自动保存游戏状态
- 保存内容：棋盘、难度、模式、时间戳
- 页面加载时自动恢复
- 音效开关独立持久化

**关键代码**：
```typescript
// app/page.tsx
// 自动保存
useEffect(() => {
  localStorage.setItem(STORAGE_KEY, JSON.stringify({
    gameState,
    difficulty,
    gameMode,
    timestamp: Date.now(),
  }));
}, [gameState, difficulty, gameMode]);

// 加载存档
useEffect(() => {
  const saved = localStorage.getItem(STORAGE_KEY);
  if (saved) {
    const parsed = JSON.parse(saved);
    setGameState(parsed.gameState);
    setDifficulty(parsed.difficulty);
    setGameMode(parsed.gameMode);
  }
}, []);
```

## 🐛 遇到的问题

### 问题 1: 落子音效不播放
**原因**：AudioContext 需在用户交互后初始化

**解决**：
- 延迟初始化 AudioContext
- 添加 `ensureContext()` 检查并恢复 suspended 状态
- 首次点击自动激活

---

### 问题 2: 动画不触发
**原因**：依赖 `animatedStones` 检查导致漏触发

**解决**：
- 改用 `lastMoveRef` 跟踪已动画的落子
- 每次 `lastMove` 变化时检查是否为新位置
- 添加 console.log 调试确认触发

---

### 问题 3: Canvas 不响应动画更新
**原因**：`drawBoard` 的 useCallback 依赖项不包含动画状态

**解决**：
- 添加 `animatedStones` 到依赖数组
- 简化动画循环，避免直接调用 `drawBoard`
- React 状态更新自动触发重绘

---

### 问题 4: 循环依赖报错
**原因**：`useEffect` 依赖 `drawBoard`，但 `drawBoard` 依赖 `animatedStones`

**解决**：
- 移除 `drawBoard` 依赖
- 纯状态驱动动画，不手动触发绘制
- 让 React 自动处理重绘
