## 0. 执行摘要

**目标：** 将五子棋 AI 强度从业余入门提升到业余高手水平

**方案选择：**
1. 短期：优化现有 MCTS（已完成）
2. 中期：重构为 α-β 剪枝（已完成）
3. 长期：接入 Rapfi 引擎（封装层已完成，等待引擎下载）

**当前进度：** 70%（等待外部依赖）

---

## 1. 问题背景

**用户反馈：**
> "比之前优化了一点，但和人下，还是有点弱"

**现状分析：**
- 当前 AI 使用 MCTS + VCF + 转置表
- 搜索深度有限（expert 仅 1500 次模拟）
- VCF 搜索仅 6 层，容易漏杀
- 评估函数简单，不懂"做棋"和埋伏

---

## 2. 方案讨论

### 方案 A - 优化现有 MCTS
| 优点 | 缺点 |
|------|------|
| 改动小 | 强度提升有限（30-40%） |
| 快速落地 | 仍有天花板 |

**决策：** 先实施，作为短期方案

### 方案 B - 重构为 α-β 剪枝
| 优点 | 缺点 |
|------|------|
| 专业架构，强度高 | 需要重写代码 |
| 搜索更深 | 思考时间增加 |

**决策：** 实施，作为中期方案

### 方案 C - 接入 Rapfi 引擎
| 优点 | 缺点 |
|------|------|
| Gomocup 第 4 名，Elo~2090 | 需要下载外部引擎 |
| C++ 优化，速度快 | 依赖二进制文件 |

**决策：** 最终方案，等待引擎下载

## 3. 实施过程

### 3.1 VCF 增强（已完成）
```
- VCF 深度：6 层 → 15 层
- 加入 VCT 连续进攻
- 新增 VCF 缓存
```

### 3.2 评估函数增强（已完成）
```
- 组合棋型：双三杀、四三杀、双四杀
- 空间控制评估
- 连接度评估
```

### 3.3 α-β 重构（已完成）
```
- α-β 剪枝 + 迭代加深
- 杀手启发 + 历史启发
- 置换表缓存
- 难度配置：深度 4/6/8/10
```

### 3.4 Rapfi 封装层（已完成）
```
- Gomocup 协议封装
- 难度对应思考时间
- 超时保护机制
```

## 4. 技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 搜索算法 | α-β 剪枝 | 比 MCTS 效率高 10 倍 |

| 终局处理 | 简化版数据库 | 平衡性能和强度 |
