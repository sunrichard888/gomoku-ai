## 🏗️ 技术实现

### 1. 皮肤配置系统 (`lib/skins.ts`)

**类型定义**：
```typescript
export interface BoardSkin {
  id: string;
  name: string;
  emoji: string;
  backgroundColor: string | string[];  // 支持单色或渐变
  gridColor: string;
  starColor: string;
  borderColor?: string;
}

export interface StoneSkin {
  id: string;
  name: string;
  emoji: string;
  blackStones: {
    gradient: [string, string, string];  // 径向渐变三色
    texture?: 'none' | 'marble' | 'wood' | 'glass';
    gloss?: number;  // 光泽强度 0-1
  };
  whiteStones: {
    gradient: [string, string, string];
    texture?: 'none' | 'marble' | 'wood' | 'glass';
    gloss?: number;
  };
}
```

- `loadSkins()` - 从 localStorage 加载皮肤偏好

- 预设皮肤数组：`BOARD_SKINS`、`STONE_SKINS`

```typescript
  const board = BOARD_SKINS.find(s => s.id === loadedSkins.board) || BOARD_SKINS[0];
  const stone = STONE_SKINS.find(s => s.id === loadedSkins.stone) || STONE_SKINS[0];
  setBoardSkin(board);
  setStoneSkin(stone);
```

```typescript
const handleBoardSkinChange = useCallback((skinId: string) => {
  const skin = BOARD_SKINS.find(s => s.id === skinId) || BOARD_SKINS[0];
  setBoardSkin(skin);
  saveSkins({ board: skinId, stone: stoneSkin.id });

  setStoneSkin(skin);
```

### 1. 棋盘渐变背景

使用 Canvas 线性渐变：
```typescript
const gradient = ctx.createLinearGradient(0, 0, BOARD_PIXELS, BOARD_PIXELS);
gradient.addColorStop(0, color1);
gradient.addColorStop(0.5, color2);

```

### 2. 棋子径向渐变

模拟立体感：
```typescript
const gradient = ctx.createRadialGradient(
  cx - radius * 0.3,  // 高光中心偏移
  cy - radius * 0.3,
  radius * 0.1,       // 内圆半径
  cx, cy,
  radius              // 外圆半径
);
```

### 3. 光泽效果（高光）

## 💡 设计决策

### 为什么使用 Canvas 而不是 CSS？
- Canvas 可以精确控制每个像素
- 渐变和光影效果更自然
- 棋子动画已经在 Canvas 中实现
- 统一渲染管线

### 为什么光泽度用 0-1 数值？
- 便于调整和未来扩展
- 可以映射到多个视觉效果
- 代码更简洁（`gloss * 0.4`）

### 为什么皮肤配置用对象数组？
- 易于添加新皮肤
- 支持动态加载
- 方便遍历渲染 UI

### 为什么棋盘和棋子分开选择？
- 自由搭配组合
- 更多个性化可能
- 符合用户直觉
